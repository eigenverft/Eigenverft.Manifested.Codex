# AGENTS.md

## Execution behavior
- For non-trivial requests, briefly structure the work before making code changes when doing so will improve clarity or execution.
- If the work naturally splits into distinct phases, chains, or concern areas, organize it into task groups when helpful.
- Keep plans concrete and action-oriented, but adapt the level of detail to the size and complexity of the work.
- Use as many tasks as needed to stay clear and effective; avoid both over-fragmenting simple work and compressing complex work too aggressively.
- When tasks are unrelated or safely independent, execute them in parallel when beneficial; otherwise, prefer focused sequential execution.
- Update the plan when scope changes, new dependencies appear, or tasks are completed, blocked, cancelled, or deferred.
- Task tracking should support execution rather than slow it down.
- For trivial, localized edits, skip formal task decomposition and execute directly.
- Before finishing, reconcile the plan so the final state of each task is clear.
